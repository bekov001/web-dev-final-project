Section B The Engine Room backend/markets/
Backend Architecture

Django REST Framework

4 models · 3 FBV · 7 CBV · JWT
I. The Schema

Four models wire the world together.

  • UserProfile — OneToOneField to User; DecimalField points for fractional payouts; auto-created via post_save signal.
  • Category — name, description, icon for filtering the front page.
  • Market — title, end_date, ForeignKey to Category; custom ActiveMarketManager; yes_probability property.
  • Trade — ForeignKey to Market & User; BooleanField choice; price_at_trade snapshot.
II. The Gatekeepers

Auth on JWT; payouts in atomic blocks.

  • SimpleJWT issues access & refresh tokens at /auth/login/ and /auth/refresh/.
  • Custom IsModeratorOrAdmin permission guards the moderation queue, approve and resolve endpoints.
  • Each trade deducts one point, snapshots the YES / NO price, and runs in transaction.atomic().
  • On resolution, winners are paid 1 / price_at_trade — a classic prediction-market payout.
III. Function-Based Views FBV
  • register_view — POST signup, returns JWT pair.
  • me_view — GET current profile with role flags.
  • trade_list_create — GET / POST trades, prices & deducts points.
IV. Class-Based Views CBV
  • MarketDetailView — full CRUD: GET / PUT / DELETE.
  • MarketListCreateView, CategoryListView — collection endpoints with ?status=active|resolved|all.
  • PendingMarketListView, MarketApproveView — approval flow.
  • AwaitingResolutionListView, MarketResolveView — settle outcomes & pay winners.

Section C The Press Room frontend/src/app/
Frontend Implementation

Angular & the broadsheet.

5 routes · HTTP service · JWT interceptor
I. Routing — app.routes.ts

Five front-page sections.

/Home — market feed
/markets/:idMarket detail & trades
/createSubmit a new market
/authSign in & register
/pendingModerator queue
II. The Wire — HTTP Layer

One service wraps every endpoint.

  • Api service uses HttpClient & returns Observable; every call piped through catchError.
  • Methods include getMarkets, createTrade, approveMarket, resolveMarket, getCurrentUser.
  • Functional HttpInterceptorFn attaches Authorization: Bearer on every request.
  • Auto-logout on 401; protected GETs are retried anonymously.
III. Control Flow
  • @if guards every auth-only block.
  • @for renders markets and trade lists.
  • @else draws the empty-state notices.
IV. Two-Way Binding
  • [(ngModel)] on the auth form fields.
  • [(ngModel)] on the create-market form.
  • Reactive validation flags errors live.
V. Event Binding
  • (click) Buy YES / Buy NO.
  • (click) Approve / reject markets.
  • (submit) Login & register flows.
